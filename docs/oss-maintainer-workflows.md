# OSS Maintainer Workflows

## Issue triage

Issues should distinguish between architecture, schema, reasoning, governance, evaluation, documentation, and security topics.

## Pull requests

Pull requests should include:

- problem statement;
- changed files;
- tests or reason tests are not applicable;
- governance or security impact;
- limitations.

## Documentation

Docs should avoid production-readiness claims and should make prototype boundaries clear.

## Sensitive data

Maintainers should reject contributions that include secrets, personal data, real client documents, or confidential legal matter materials.
