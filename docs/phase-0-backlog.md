# Phase 0 Backlog

This backlog turns the concept into engineering tasks. It is intentionally more detailed than the GitHub issue seed list.

## P0. Must exist before serious implementation

- Universal core model spec.
- Contractual package manifest.
- Bootstrap JSON schema.
- Management Plane policy matrix.
- Governance profiles per candidate type.
- Translation Layer artifact model.
- Benchmark and red-team scenario schemas.
- Synthetic decision trace example.

## P1. First coded prototype

- Build a Source Graph from synthetic legal sources.
- Build a Case Graph from `examples/contract_supply_case.json`.
- Attach source-grounded claims to the case.
- Validate a reviewed bootstrap JSON object.
- Run a deterministic placeholder formal check.
- Propose a candidate principle.
- Apply a governance profile.
- Export a decision trace containing knowledge and policy version coordinates.

## P2. First meaningful legal package work

- Expand contractual vocabulary.
- Add authority model rules for general vs special contractual norms.
- Add temporal rules for obligation formation and performance date.
- Add contradiction taxonomy examples.
- Add legal operators for contractual counterfactuals.
- Add 5-10 benchmark tasks.
- Add 5-10 red-team scenarios.

## P3. OSS readiness

- Add `CONTRIBUTING.md`.
- Add issue labels.
- Add project board or milestone structure.
- Add a short architecture diagram.
- Add maintainer workflow for reviewing institutional package contributions.
- Add security policy for secrets and sensitive legal materials.
